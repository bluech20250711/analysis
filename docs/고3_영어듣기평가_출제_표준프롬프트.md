이 문서는 두 개의 파트로 구성됩니다.

2. **마스터 생성 프롬프트** — Gemini/Claude API 등에 그대로 투입할 수 있는 프롬프트 템플릿 (변수: `{TOPIC}`, `{DIFFICULTY}`, `{COUNT}` 등)

## 1. 마스터 생성 프롬프트 (전체 세트 17문항 생성용)

```
당신은 대한민국 고3 전국연합학력평가(모의고사) 영어 영역 듣기평가 출제 전문가입니다.
2025학년도·2026학년도 3월 고3 전국연합학력평가 듣기평가의 문체, 구조, 난이도를 학습된
표준으로 삼아 새로운 듣기평가 1~17번 세트를 출제하십시오.

[전체 원칙]
- 총 17문항, 각 문항은 아래 "문항별 상세 규격"의 화자 구조·형식·길이·오답 설계 원칙을 정확히 따를 것.
- 어휘 수준: 고3 학력평가 표준 수준(전문용어 최소화, 실생활 밀착 소재).
- 인물 이름은 매번 다르게 하되 흔한 영미권 이름(Jake, Emma, Andrew, Sophia, David 등) 사용.
- 브랜드/프로그램/장소명은 실재하지 않는 가상의 고유명사로 생성 (예: "Harmony Rental Shop",
  "DNA Discovery Program" 같은 톤).
- 모든 대화문은 자연스러운 구어체 영어로 작성하고, 화자 표시는 M:/W: 로 통일.
- 각 문항마다 "정답"과 "선택지 5개(①~⑤)"를 함께 생성하되, 대본(스크립트)과 문제지 선택지를
  분리하여 출력할 것.
- 오답 선택지는 스크립트 내용과 표면적으로 관련되어 보이지만 명확히 틀리도록 설계
  (숫자/조건을 살짝 바꾸거나, 언급되지 않은 정보를 넣거나, 반대 상황을 서술).
- 3점 배점 문항(6, 11 또는 13, 15)은 계산이 필요하거나 대화 맥락 추론이 한 단계 더 필요하도록
  난이도를 살짝 높일 것.
- 16~17번은 반드시 하나의 지문(여자 단독 강의체)에서 2문항을 함께 출제.

[출력 형식]
아래 순서로 출력하십시오.
1) 문제지용 선택지 (1번~17번, 국문 지시문 + 선택지)
2) 듣기 대본 전체 (1번~17번, 화자 표시 포함, 실제 방송 대본 형태)

[생성 파라미터]
- 주제 영역: {TOPIC_POOL}  (예: 학교생활, 지역사회 행사, 건강, 환경, 기술, 취미/여가 중 랜덤 배분)
- 전체 난이도: {DIFFICULTY}  (기본값: 2025/2026 3월 학평과 동일 수준)
- 회차명: {EXAM_TITLE}  (예: "2027학년도 O월 고3 전국연합학력평가")
```

## 2. 문항별 상세 규격 (Few-shot 근거 포함)

### 1번 — 목적 파악 (2점)
- **화자 구조**: 남자 단독 발화 (기관 안내/방송)
- **구조**: 인사(Hi/Hello) → 화자 소속·역할 소개 → 배경 상황 설명(2~3문장) → 핵심 목적 문장(주로 후반부) → 마무리 인사
- **길이**: 5~7문장
- **소재 예시**: 체육관 이용수칙 안내, 오디션 프로그램 투표방식 변경 공지
- **선택지 설계**: 5개 모두 "~하려고" 형태의 한국어, 목적의 카테고리(공지/모집/설명/권장/안내)를 다르게 배치하되 배경상황에서 언급된 소재 어휘를 재활용해 혼동 유도

### 2번 — 여자의 의견 (2점)
- **화자 구조**: 대화 (교사-학생 또는 부부/가족)
- **구조**: 남자가 상황/문제 제기 → 여자가 되묻기 → 여자의 진단/조언 → 남자의 반박이나 변명 → 여자가 조언 재강조 → 남자 수긍
- **길이**: 8~10턴
- **선택지 설계**: 대화의 표면적 소재(예: 필기, 구독서비스)를 활용한 5개 한국어 문장, 정답은 여자가 반복해서 강조한 조언과 정확히 일치해야 함

### 3번 — 요지 (2점)
- **화자 구조**: 남자 단독 발화 (라디오/유튜브 방송 톤)
- **구조**: "Hi/Greetings, this is [이름] from [채널/프로그램명]" → 통념(많은 사람이 오해하는 것) 제시 → 반박 논리 → 근거(연구/경험) → 결론 강조
- **길이**: 6~8문장
- **선택지 설계**: 정답은 결론 문장의 요지를 한국어로 정확히 반영, 오답은 소재는 같지만 방향이 반대이거나 지나치게 좁은/넓은 범위

### 4번 — 그림 불일치 (2점)
- **화자 구조**: 대화 (사진 또는 장면을 함께 보며 묘사)
- **구조**: 장면 설명 시작 → 시각 요소 5개를 순서대로 하나씩 언급 (배경/소품1/인물/소품2/기타) → 각 요소마다 상대방 반응
- **길이**: 8~9턴
- **주의**: 스크립트 자체는 모든 요소가 논리적으로 일치하게 서술하고, 실제 오답(그림 불일치)은 이미지 생성 단계에서 텍스트와 다르게 그려서 만들어야 함 (텍스트 생성만으로는 정답 요소를 표시하기 어려우므로, 이미지 연동 시 "① 요소가 스크립트 설명과 다르게 그려짐" 같은 메타 정보를 별도로 출력)

### 5번 — 여자가 할 일 (2점)
- **화자 구조**: 대화 (행사 준비 점검)
- **구조**: 행사 언급 → 준비물/역할 4~5개를 하나씩 점검(완료/미완료 확인) → 마지막에 깜빡한 일 하나 발견 → 여자가 그 일을 맡기로 함
- **길이**: 8~9턴
- **선택지 설계**: 대화 중 언급된 모든 준비 항목을 선택지로 만들고, 정답은 마지막에 여자가 "하겠다"고 명시한 항목

### 6번 — 지불 금액 (3점)
- **화자 구조**: 대화 (문의-계산형)
- **구조**: 상품/서비스 문의 → 옵션 A/B 가격 제시 → 부가서비스 포함여부 확인 → 추가비용(세션당 요금 등) → 할인조건(신규가입/장기결제) → 최종 합계
- **길이**: 8~9턴
- **계산 원칙**: 반드시 "기본가 + 부가비용 − 할인" 3단계 연산이 되도록 설계, 선택지는 각 단계에서 하나씩 놓쳤을 때 나오는 오답 금액 포함
- **예시 산식**: (세션 패키지 가격) + (세션당 부가비용 × 세션수) − (신규가입 할인) = 정답

### 7번 — 이유 파악 (2점)
- **화자 구조**: 대화 (오답 소거형)
- **구조**: 여자(또는 남자)가 어떤 일을 할 수 없다고 말함 → 상대가 그럴듯한 이유 2~3개를 추측 → 매번 "아니야" 부정 → 마지막에 진짜 이유 공개
- **길이**: 8턴
- **선택지 설계**: 대화 중 언급되었다가 부정된 이유들을 오답으로, 마지막에 공개된 진짜 이유를 정답으로

### 8번 — 언급되지 않은 것 (2점)
- **화자 구조**: 대화 (정보 소개형)
- **구조**: 상대가 모르는 장소/서비스를 소개 → 종류(취급품목) → 위치 → 비용 → (수령방법 또는 기타) → 영업시간/일시, 이 중 5개 카테고리 중 정확히 4개만 언급
- **길이**: 8~9턴
- **선택지 설계**: 5개 카테고리를 선택지로 제시, 대화에 등장하지 않은 카테고리가 정답

### 9번 — 안내문 불일치 (2점)
- **화자 구조**: 여자(또는 남자) 단독 발화, 프로그램/이벤트 안내
- **구조**: 프로그램 소개 → 대상(연령/자격) → 기간/일정 → 진행방식 → 비용/혜택 → 신청방법, 5개 조항을 순서대로 언급
- **길이**: 6~7문장
- **선택지 설계**: 5개 선택지 중 4개는 원문 그대로, 1개는 숫자/조건을 미묘하게 변형(예: 원문 "첫 10명만 개별 피드백" → 선택지 "모든 참가자가 개별 피드백")

### 10번 — 표 보고 고르기 (2점)
- **화자 구조**: 대화 (표를 보며 조건 소거)
- **구조**: 5개 모델이 있는 표 제시 → 예산 조건으로 1개 소거 → 속성 조건(크기/재질 등)으로 1개 소거 → 속성 조건(등급/평가 등)으로 소거 → 마지막 선호 조건(색상/방수 등)으로 최종 1개 확정
- **길이**: 8~9턴
- **표 설계 원칙**: 열은 4~5개(가격, 속성1, 속성2, 속성3), 행은 5개(A~E), 마지막까지 2개가 남았다가 최종 조건으로 갈림

### 11번 / 12번 — 마지막 말 응답 (단문형, 2~3점)
- **화자 구조**: 대화 3~4턴
- **구조**: 상황 제시 → 문제 발생 → 상대의 제안/질문 → (마지막 화자 대사 이후 응답 선택)
- **길이**: 3~4턴 (13·14번보다 짧음)
- **소재 예시**: 도서관 충전좌석 찾기, 사진 공모전 참가 권유
- **선택지 설계**: 5개 모두 자연스러운 영어 응답, 정답은 직전 대사의 정보(예: "스마트폰 부문 있다")에 논리적으로 이어지는 반응

### 13번 / 14번 — 마지막 말 응답 (장문형, 2~3점)
- **화자 구조**: 대화 8~9턴
- **구조**: 배경 상황 설명 → 문제/불편 언급 → 대안 제시 및 설명(작동 방식 등 세부사항 포함) → 상대의 이해/동의 → (마지막 대사 이후 응답 선택)
- **길이**: 8~9턴 (11·12번보다 김, 3점 배점일 경우 세부 정보가 2개 이상 겹쳐야 함)
- **소재 예시**: 제로웨이스트 매장 이용법, 청바지 업사이클링

### 15번 — 상황설명 (3점)
- **화자 구조**: 남자 내레이터 단독 발화 (항상 3인칭 서술)
- **구조**: 인물 A(조언을 구하는 사람)와 인물 B(조언자)의 관계 소개 → A의 상황/문제 서술 → B가 관찰한 원인 → B가 하려는 조언의 내용(직접 대사 없이 서술) → 마지막 문장 "In this situation, what would [B] most likely say to [A]?"
- **길이**: 6~7문장
- **선택지 설계**: 정답은 서술된 B의 조언 내용을 자연스러운 영어 대사로 표현한 것, 오답은 소재는 같지만 반대 방향이거나 무관한 조언

### 16~17번 — 장문 듣기 (주제 + 언급되지 않은 것, 각 2점, 2회 청취)
- **화자 구조**: 여자 단독 발화 (강의/설명체)
- **구조**: 인사 및 주제 도입("Today, we'll talk about~") → 4개의 병렬적 사례를 "First, ~ / Next, ~ (또는 We also have~) / Now/Finally, ~"로 순서대로 소개, 각 사례마다 1~2문장의 설명 → 마무리 문장
- **길이**: 8~10문장, 사례 4개 고정
- **16번(주제) 선택지**: 영문 5개, 전체 내용을 아우르는 추상화 수준으로 설계 (너무 좁거나 너무 넓은 오답 포함)
- **17번(언급 안 된 것) 선택지**: 실제 언급된 4개 사례 + 언급되지 않은 1개(정답)를 병기, 5개 모두 같은 상위 카테고리(동물/재료 등)에서 선택

## 3. 오답 설계 공통 체크리스트 (모든 문항 공통 적용)

생성 후 아래 항목을 자체 점검하도록 프롬프트에 포함할 것:

1. 정답 근거가 스크립트 내 명시적 문장/숫자로 뒷받침되는가?
2. 오답들이 스크립트의 다른 부분(다른 인물의 말, 다른 시점, 다른 조건)에서 유래해 매력적인 오답인가?
3. 정답이 스크립트 맨 마지막 부분에 있어 "끝까지 들어야" 풀리는 구조인가? (4·6·7·9·10번 특히 중요)
4. 배점 3점 문항은 2단계 이상의 정보 결합(계산/추론/맥락종합)이 필요한가?
5. 선택지 5개의 문장 길이와 문법 구조가 서로 비슷한가? (특정 선택지만 유독 길거나 짧지 않은가)

## 4. 활용 팁 (ExamBuilder 연동 시)

- 문항 단위 재생성이 필요할 때는 "2. 문항별 상세 규격"의 해당 블록 + "3. 공통 체크리스트"만 잘라서 API에 전달하면 됩니다.
- 4번(그림 불일치)은 텍스트 생성만으로는 정답을 만들 수 없으므로, 스크립트 생성 후 "그림 요소 리스트"를 별도 JSON으로 뽑아 이미지 생성 프롬프트에 넘기고, 그중 1개 요소를 의도적으로 다르게 그리는 후처리 단계가 필요합니다.
- 6번(계산형)은 숫자 오류를 방지하기 위해 생성 후 별도의 계산 검증 스텝(간단한 산식 재계산)을 두는 것을 권장합니다.
- 대본과 문제지를 분리 출력하도록 프롬프트에 명시해두면, 학생용 문제지 PDF와 교사용 대본 PDF를 각각 뽑아내는 기존 워크플로우(HWPX/DOCX 변환)에 바로 연결할 수 있습니다.
